Welcome new developers!

If you are wanting to create a personal web page using GitHub, then follow the instruction below!

Step 1

Set up a GitHub account at GitHub

step 2

Set up a repository for your website / project by selecting the profile circular icon and drop down in the top right hand corner.

step 3

In the dropdown window, select your repositories.

step 4

Now select the green “new” button in the top right portion of the site.

step 5

Give your repository a name in the “repository name” box near the top.

Note that at this point you can add a README.md by checking the “Add a README file” or you can opt not to do so and add it later but a README.md will be required for the repository to work correctly.

step 6

Once you are satisfied you can hit the green “Create Repository” button towards the bottom.

step 7

You will need to create the files listed below. You can do this within GitHub repository by selecting the “Add file” drop down. Here you can create them directly in GitHub, upload them from a local device or even drag and drop them into the repository screen. Finally, you can also create the files locally then push them to GitHub by using git (add, commit -m and push origin main)

README.md : if not created at time of repository creation

index.html

style.css (optional: used to help with site formatting)

style.css in the index.html file make sure that you include a link to the style.css file in the element using the following format:

between pointy brackets include: link rel = "stylesheet" href = "style.css"

This will allow you to easily make a lot of changes to the entire html document.

This file must have the .css extension and not contain and html code.

logic.js (optional)

step 8

commit the files after adding them by clicking the green “Commit new file” button at the bottom.

step 9

While in the repository add a new environment by clicking “ settings at the right of the bar under your user name and repositor name.

step 10

Next select “Environments” on the left side under “Code and automation.”

step 11

Click the gray “New environments” button on the right, give the environment a name then configure it by clicking the green “Configure environment.”

step 12

Now, under settings, click “pages” on the left side under “code and automation”

step 13

Under “Branch,” select and ensure “main” is selected, then click the “save” button next to it.

step 14

Finally, you can refresh the browser and your website will show towards the top under “GitHub Pages,” there you will find a “button that says “Visit site” you can click that and it will show your website.

Congradulations!
